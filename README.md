# Enterprise AI Document Ingestion Pipeline 🚀

Мощная интеллектуальная система уровня Enterprise для конвертации смешанных PDF, DOC, DOCX документов (сканы, таблицы, текст) в структурированный формат и интеграции в системы LLM RAG (Retrieval-Augmented Generation). Оснащена гибридным поиском, продвинутой очисткой шума и семантическим чанкингом.

---

## 🏗️ Архитектура: Enterprise Hyper-RAG Search Layer

В проекте реализован **Гибридный поиск (Hybrid Search Algorithm)**, устраняющий главный недостаток LLM (плохое распознавание цифр, имен и редких терминов).

### 🔍 Search Pipeline

1. **User Query (Векторный Анализ):** Пользователь формулирует вопрос, например, "Какая стоимость тарифа Бизнес?".
2. **Query Rewriting:** Запрос переписывается для векторного поиска — к нему добавляются синонимы, извлекаются ключевые сущности, чтобы максимально раскрыть смысл запроса `(semantic normalization)`.
3. **Hybrid Search Routing:**
   - **Vector Search (ChromaDB):** Использует LLM-эмбеддинги для поиска документов, близких *по смыслу* (поиск концептов, абстракций, схожих паттернов).
   - **BM25 Keyword Search (`rank_bm25`):** Алгоритм статистического ранжирования для точного поиска редких терминов *по ключевым словам*, совпадений ФИО и конкретных числовых метрик.
4. **Hybrid Ranking Algorithm (Ранжирование):**
   - Баллы нормализуются от 0 до 1. 
   - `final_score = (vector_score * 0.6) + (bm25_score * 0.4)`
5. **Metadata Filtering:** Фильтрация результатов по метаданным (конкретная глава `section_filter`, источник `source_filter`, страница).
6. **Context Builder (Deduplication & Merge):** Топ-К (top-K) фрагментов (чанок) отправляются в сборщик контекста для LLM. Соседние чанки "склеиваются", дубликаты убираются. Если контекст превысил лимит, применяется Context Compression.
7. **LLM Answer:** Сформированный `llm_context_prompt` и вопрос отдаются в LLM (`g4f` / `OpenAI`) для генерации точного ответа с цитированием страниц документа.

## 🌟 Возможности Конвертера (Ingestion)

1. **Многоформатность:** Поддержка PDF (Vector + Scans), `DOC`, `DOCX` (С автоматической конвертацией старых форматов через Python win32com).
2. **Advanced Smart Cleaner (Удаление мусора):**
   - Детектирует и удаляет "сквозные" колонтитулы (найденные на >60% страниц).
   - Извлекает URL-ссылки статей в корень JSON-объекта (автоматическое сохранение `article_url`).
   - Очищает водяные знаки, даты выгрузки, счетчики страниц и подписи пользователей.
3. **Contextual Structure Building:** 
   Объединяет разорванные предложения, дедуплицирует повторяющиеся куски текста, восстанавливает структуру (разделы `sections`).
4. **Table-Aware Retrieval:** Таблицы полностью парсятся с сохранением рядов и колонок (Array) и отправляются в базу как отдельные проиндексированные чанки.
