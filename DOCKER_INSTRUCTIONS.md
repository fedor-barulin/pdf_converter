# Инструкция по запуску проекта через Docker

Данный проект поддерживает запуск в изолированных контейнерах с помощью **Docker** и **Docker Compose**. Это позволяет избежать проблем с установкой зависимостей (Python, Node.js, Tesseract OCR) на локальную машину.

## Предисловие (Важно)
В Docker-контейнере используется Linux-образ. Функция конвертации старых форматов `.doc` в `.docx` (использующая Microsoft Word через библиотеку `pywin32`) в Docker **работать не будет**, так как она требует установленного MS Word и Windows. Основной функционал (PDF, DOCX, парсинг, RAG) работает корректно.

---

## 1. Подготовка
Убедитесь, что у вас установлены:
- **Docker Desktop** (для Windows/Mac) или **Docker Engine** + **Docker Compose** (для Linux).

## 2. Сборка и запуск
Откройте терминал в корневой директории проекта и выполните команду:

```bash
docker-compose up --build
```

- `--build` — принудительно пересоберет образы (нужно при первом запуске или изменении зависимостей).
- Без флага `-d` вы будете видеть логи всех сервисов в реальном времени. Если хотите запустить в фоновом режиме, используйте `docker-compose up -d --build`.

## 3. Доступ к приложению
После успешного запуска проект будет доступен по следующим адресам:

- **Frontend (Веб-интерфейс):** [http://localhost:3000](http://localhost:3000)
- **Backend (API):** [http://localhost:8000](http://localhost:8000)
- **Документация API (Swagger):** [http://localhost:8000/docs](http://localhost:8000/docs)

## 4. Полезные команды

### Остановка проекта
Чтобы остановить контейнеры, нажмите `Ctrl+C` (если запущен не в фоновом режиме) или выполните:
```bash
docker-compose stop
```

### Полное удаление контейнеров
Если нужно очистить ресурсы:
```bash
docker-compose down
```

### Просмотр логов
```bash
docker-compose logs -f
```

---

## Что внутри Docker?
- **Backend:** Образ на базе Python 3.14-slim. Внутри установлены все необходимые библиотеки, включая `Tesseract OCR` для распознавания текста и `poppler-utils` для обработки PDF.
- **Frontend:** Образ на базе Node.js 18. При сборке выполняется `npm run build`, а затем приложение отдается через статический сервер `serve`.
